## 2026-07-22 — v0.2 라이브 렌더 검증 후 수정

GitHub 커밋 → raw.githack 렌더로 **실제 브라우저 검증**(데스크톱 1293×715 / 모바일). 정적 검사로는 못 잡은 3건 발견·수정.

1. **로고 사각형 헤일로** — 알파 키잉이 JPEG 노이즈까지 살려 로고 주변에 옅은 사각 veil이 남음(트림 bbox 946×1082). 
   → 알파 임계 리맵 `clip((t-0.14)/0.86)` 적용, 배경부 최대 알파 0.0 확인. 트림 결과 797×847로 축소 = veil 실재 확인.
2. **데스크톱 히어로가 첫 화면을 넘김** — 실측 히어로 텍스트 920px vs 뷰포트 715px → **CTA 버튼이 144px 아래로 잘림**(전환 요소가 첫 화면에 없음). 
   → 로고 152→128px, h1 3.35→2.75rem(+`text-wrap:balance`), 상하 패딩·CTA 마진 축소.
3. **work-01 워터마크 잔존** — 상단 88% 크롭이 부족해 "→비포사진은 다음장으로"가 갤러리에 그대로 노출. → 80% 크롭으로 재생성.
4. (미세) 아카이브 밴드 이미지 opacity .72 → .82.

**검증 통과**: 가로 스크롤 0 · 모바일 퀵바 정상 · 모바일에서 리빌 opacity 1(숨김 없음) · 갤러리/메뉴/정책/FAQ/방문 레이아웃 이상 없음.

### v0.2 추가 수정 — 히어로 높이의 진짜 원인
h1·로고·패딩을 줄여도 CTA가 91px 밖에 남아 원인을 다시 측정. **텍스트가 아니라 히어로 이미지가 범인**이었다.
데스크톱에서 `.hero-img{height:100%}`가 auto 높이 행 안에서 해석되지 않아 `<img>`가 원본 비율(1400×1867)대로 950px까지 늘어나 
행 전체를 1016px로 밀어올림. → `.hero-img{position:relative;height:auto}` + `img{position:absolute;inset:0;object-fit:cover}`로 
이미지가 행 높이에 기여하지 않게 변경. **결과: 히어로 = 정확히 뷰포트 1개(710px), CTA가 화면 안에 95px 여유로 노출.**
교훈: "요소가 화면을 넘친다"는 증상에서 텍스트만 줄이지 말고 **어느 자식이 행 높이를 만드는지 먼저 실측**할 것.
